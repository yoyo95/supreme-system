# supreme-system
# Scripts de maintenance et d’administration

Ce dépôt GitHub est fourni par [joseph](https://jzerbib.fr/)

Il a pour objectif de présenter et centraliser différents scripts destinés à améliorer les capacités de maintenance, d’administration, d’automatisation et de diagnostic d’un environnement informatique.

Les scripts présents dans ce dépôt sont conçus pour assister les administrateurs système, techniciens ou responsables informatiques dans leurs tâches quotidiennes.

---

## Objectif du dépôt

Ce dépôt regroupe des scripts permettant notamment de :

- automatiser des tâches répétitives ;
- améliorer la maintenance des postes, serveurs ou services ;
- faciliter le diagnostic technique ;
- accélérer certaines opérations d’administration ;
- standardiser des procédures internes ;
- réduire les erreurs humaines lors d’opérations récurrentes ;
- documenter des méthodes d’intervention techniques.

Chaque script a pour but d’augmenter les capacités de maintenance, de supervision ou d’exploitation d’un environnement informatique.

---

## Contenu du dépôt

Chaque dossier ou fichier correspond à un usage spécifique.

Exemples de catégories possibles :

- scripts d’administration système ;
- scripts de diagnostic ;
- scripts d’automatisation ;
- scripts liés à Microsoft 365 ;
- scripts réseau ;
- scripts de gestion de postes utilisateurs ;
- scripts de maintenance applicative ;
- scripts de reporting ou d’inventaire.

---

## Utilisation

Avant toute exécution, il est recommandé de :

1. lire le contenu du script ;
2. comprendre son objectif ;
3. vérifier les prérequis techniques ;
4. tester le script dans un environnement de test ;
5. adapter les variables si nécessaire ;
6. exécuter le script avec les droits appropriés.

Certains scripts peuvent nécessiter des droits administrateur ou des permissions spécifiques sur les systèmes ciblés.

---

## Prérequis

Selon les scripts, les prérequis peuvent varier.

Ils peuvent inclure :

- Windows PowerShell ou PowerShell 7 ;
- Python ;
- droits administrateur local ;
- accès réseau ;
- modules spécifiques ;
- compte Microsoft 365 avec permissions adaptées ;
- accès API ;
- environnement Windows Server ou poste client Windows.

Les prérequis spécifiques doivent être vérifiés dans chaque script ou dans sa documentation associée.
